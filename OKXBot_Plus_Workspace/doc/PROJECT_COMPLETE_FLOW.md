# CryptoOracle Plus 全流程与交易逻辑详解 (v3.3.12)

本文档详细描述了 **CryptoOracle Plus (v3.3.12)** 的系统架构、执行流程及核心交易逻辑。作为高频/中高频量化交易系统，本项目采用了 **SOA (面向服务架构)** 和 **双频轮询 (Dual-Frequency Polling)** 机制，以在“低延迟止损”和“抗市场噪音”之间取得完美平衡。

---

## 1. 系统架构 (System Architecture)

系统基于 Python `asyncio` 异步框架构建，分为三层服务架构：

### 1.1 核心服务层 (Service Layer)
*   **🧠 策略大脑 (`src/services/strategy`)**:
    *   负责对接 DeepSeek LLM。
    *   根据 K 线形态、技术指标 (RSI/MACD/ADX) 生成交易信号。
    *   动态调整 AI "人格" (激进/稳健/避险)。
*   **⚔️ 执行引擎 (`src/services/execution`)**:
    *   负责与交易所 (OKX) 交互。
    *   计算技术指标 (Indicators)。
    *   执行交易前风控 (滑点保护、微利拦截、震荡保护)。
    *   下单与状态追踪。
*   **🛡️ 全局风控 (`src/services/risk`)**:
    *   负责账户级资金管理。
    *   监控 PnL (盈亏) 曲线。
    *   执行硬止损 (Hard Stop Loss) 和 全局止盈。

### 1.2 双频轮询机制 (Dual-Frequency Architecture)
为了解决“长轮询导致止损不及时”与“短轮询导致噪音干扰”的矛盾，v3.3.12 引入了双层循环：

1.  **低频策略循环 (Strategy Loop)**
    *   **频率**: `30s` - `60s` (可配)。
    *   **职责**: 获取完整 K 线 -> 计算指标 -> 请求 AI -> 决策开平仓。
    *   **目的**: 过滤 1m 周期内的微小噪音，让 AI 捕捉更确定的趋势。

2.  **高频看门狗循环 (Watchdog Loop)**
    *   **频率**: 固定 `5s`。
    *   **职责**: 仅获取当前价格 -> 检查持仓浮亏 -> 触发硬止损。
    *   **目的**: 消除止损盲区。即使 AI 在思考，看门狗也能在暴跌 5秒内强制平仓。

## 2. 详细执行流程 (Detailed Execution Flow)

### 2.1 启动阶段 (Startup)
1.  **环境检查**: 验证 API Key、代理连通性、DeepSeek 响应速度。
2.  **资产盘点**:
    *   读取 `config.json` 中的 `initial_balance`。
    *   扫描账户权益，检测“闲置资金”或“异常充值”。
    *   **智能基准 (Smart Baseline)**: 如果发现账户余额 > 配置本金，自动记录 `deposit_offset`，确保 PnL 从 0 开始计算。
3.  **数据预热**: 并发请求所有监控币种的历史 K 线，预计算 MACD/EMA 指标。

### 2.2 运行阶段 - 看门狗 (High-Frequency)
每 5 秒执行一次：
1.  **快速询价**: 调用 `fetch_ticker` 获取最新成交价。
2.  **硬止损检查**:
    *   计算当前持仓 PnL。
    *   如果 `PnL < max_loss_rate` (如 -15%)，**立即触发市价全平**。
    *   **不经过 AI**，不进行任何犹豫。

### 2.3 运行阶段 - 策略 (Low-Frequency)
每 `interval` 秒执行一次：
1.  **行情获取**: 获取最新 OHLCV K 线数据。
2.  **指标计算**: 更新 RSI, MACD, Bollinger Bands, ADX。
3.  **市场状态判定**:
    *   `ADX > 30`: **HIGH_TREND** (单边趋势)。
    *   `ADX < 20`: **LOW** (窄幅震荡)。
    *   `Volatility > 0.5%`: **HIGH_CHOPPY** (剧烈波动)。
4.  **AI 分析**:
    *   根据状态选择 Prompt (激进/稳健/网格)。
    *   投喂数据：K线、指标、持仓状态、账户余额。
    *   获取信号：`BUY` / `SELL` / `HOLD` 及 信心 `HIGH`/`MED`/`LOW`。
5.  **交易执行 (Execution)**:
    *   **信心过滤**: 如果 `Confidence < min_confidence`，强制 HOLD。
    *   **滑点检查**: 如果当前价与 K 线收盘价偏差 > 1%，跳过交易。
    *   **微利拦截**: 如果是平仓且利润微薄 (不足以覆盖手续费)，拦截 (除非 AI 信心为 HIGH)。
    *   **震荡保护 (Anti-Whipsaw)**: 如果持仓时间 < 45s 且浮亏微小，强制持有，防止反复割肉。
    *   **下单**: 计算数量 (张数/币数转换) -> 发送订单 -> 推送通知。

## 3. 核心交易逻辑 (Core Trading Logic)

### 3.1 信号生成 (Signal Generation)
AI 不是盲目输出，而是基于 **Prompt Engineering** 的角色扮演：
*   **趋势市 (Trend)**: 鼓励顺势突破，忽略回调噪音。
*   **震荡市 (Choppy)**: 严禁追涨杀跌，优先观望，寻找布林带反转。
*   **垃圾时间 (Low Vol)**: 禁止频繁开仓，等待变盘。

### 3.2 资金管理 (Fund Management)
*   **固定本金模式**: 严格按照 `initial_balance` 运行，哪怕账户里有 100万U，配置了 1000U 就只用 1000U。
*   **配额隔离 (Quota Isolation)**:
    *   每个币种分配固定比例 (如 50%)。
    *   防止单币种亏损拖累整个账户。
*   **激进模式 (Aggressive Mode)**:
    *   当 AI 信心为 `HIGH` 时，允许突破单币种配额，调用账户中最多 **90%** 的闲置资金进行重仓出击。

### 3.3 关键风控机制 (Risk Mechanisms)
1.  **冷却机制 (Cooldown)**:
    *   **绝对冷却 (3s)**: 防止 API 连发。
    *   **普通冷却 (30s)**: 防止 AI 在 1分钟内反复反手。
2.  **微利拦截 (Profit Filter)**:
    *   平仓门槛 = `Taker费率 * 2 + 0.05%`。
    *   防止“赚了指数亏了手续费”。
3.  **震荡磨损保护 (Anti-Whipsaw)**:
    *   **规则**: 持仓 < 45s 且 PnL > -0.6%。
    *   **动作**: 强制 HOLD。给交易一点“呼吸空间”，避免被 K 线上下影线扫地出门。

## 4. 常见问题 (FAQ)

*   **Q: 为什么日志显示 "SKIPPED_PROFIT"?**
    *   A: AI 想要平仓，但当前利润不足以覆盖手续费，且信心不是 HIGH。系统帮你拿住了单子。
*   **Q: 为什么日志显示 "SKIPPED_WHIPSAW"?**
    *   A: 刚开仓不到 45秒，AI 可能因为价格波动想跑，但系统强制要求多拿一会儿，防止频繁割肉。
*   **Q: 机器人反应慢怎么办？**
    *   A: 检查 `config.json` 的 `loop_interval`。但请注意，v3.3.12 已有看门狗保护，策略慢一点反而是好事（更稳）。
